# GTM Pipeline Health Crew

A CrewAI **Flows + Crews** system that audits a B2B sales pipeline, identifies at-risk deals, and produces a manager-ready intervention brief — with a human approval gate before anything is published.

Built as a deliberate architectural counterpart to my [AgenticAI LangGraph project](https://github.com/nmanthas): **same problem space (GTM operations), two orchestration philosophies.**

## What it does

```
PipelineHealthFlow (deterministic backbone)
│
├─ @start   ingest_pipeline        # load + rule-score 20 opportunities (no LLM)
├─ @listen  run_analysis_crew      # Crew 1: Risk Analyst → Win-Pattern Researcher
├─ @router  triage                 # routes on typed state, not LLM text
│     ├─ "escalate"  → run_advisory_crew   # Crew 2: Deal Desk Advisor → Comms Writer
│     │                → human_review      # approve / edit / reject (HITL gate)
│     │                → publish           # write outputs/pipeline_brief.md
│     └─ "summarize" → log_healthy_pipeline
```

See [`outputs/sample_brief.md`](outputs/sample_brief.md) for example output, and `flow_diagram.html` for the auto-generated flow visualization.

## Design principles

1. **Deterministic where possible, LLM where valuable.** Risk scoring is a pure rule engine (`src/tools/pipeline_tools.py`) — reproducible, auditable, cheap. Agents interpret and advise on top of the scores; they never compute them. Routing decisions read typed Pydantic state, never LLM prose.
2. **Gradual autonomy.** v1 runs at 100% human review (`human_review` step). The gate is designed to be relaxed incrementally — e.g. auto-approve briefs under a deal-count threshold — as the system earns trust. Autonomy is earned, not assumed.
3. **Governance-aware recommendations.** The Deal Desk Advisor's playbook respects pricing guardrails: deals over the discount threshold get an *approval path*, never a bigger discount.
4. **Swap-ready data layer.** `load_pipeline()` reads a CSV; replace it with a `simple-salesforce` query or a Salesforce MCP call and nothing downstream changes.

## Why CrewAI here, LangGraph there

| Dimension | AgenticAI (LangGraph) | This repo (CrewAI) |
|---|---|---|
| Orchestration | Explicit graph with a custom router node | Role-based Crews inside an event-driven Flow |
| State | Checkpointer-backed, resumable mid-graph | Typed Pydantic Flow state |
| Control vs. velocity | Maximum control, more code | Fast to ship, readable by non-specialists |
| Human-in-the-loop | Interrupt-based, resumable | Explicit gate step in the Flow |
| Best fit | Long-running, auditable, complex branching | Structured business workflows with clear roles |

The choice of orchestration framework is an architectural decision, not a preference — framework choice can move agent benchmark performance by double-digit percentage points on identical models. This repo exists to demonstrate both sides of that trade-off on one real GTM problem.

## Run it

```bash
pip install "crewai[tools]"
export ANTHROPIC_API_KEY=sk-ant-...   # or configure any litellm-supported provider

python -m src.flow          # full run with human review gate
python -m src.flow --plot   # generate flow_diagram.html only (no LLM calls)
```

Default model is `anthropic/claude-sonnet-4-6`, configurable per agent in `src/crews/*.py`.
